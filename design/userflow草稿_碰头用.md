## 三、偏好学习曲线 — "3 次之后 hands-free"

```
第 1 次 ──── 完整审查（~20 分钟人工）
              AI 学到：你的删减风格、保留偏好
              
第 2 次 ──── 轻度审查（~10 分钟）
              AI 预删减更准，你只改几处
              
第 3 次 ──── 快速确认（~5 分钟）
              "看一眼章节摘要 + 听一遍预览 + 导出"
              
第 4 次+ ─── Hands-free 模式（可选）
              "上传 → 等 5 分钟 → 拿走成品"
              AI 自动应用已学到的偏好
              用户可以选择跳过审查直接出片
```

## 四、两类用户的 Flow 差异

### A. 个人主播 / 个人剪辑师

```
痛点：每期花大半天听完整录音找要删的部分
目标：快速出粗剪 + 高光片段，发小宇宙/抖音

Flow 重点：
· 简单 — 上传就跑，不要太多设置
· 审查要直觉 — 看章节摘要就知道删了什么
· 高光自动推荐 — 选 3 个片段自动拼片头
· 偏好学习快速生效 — 第 2 次就明显省时间
```

### B. 机构 / MCN 专业剪辑师

```
痛点：同时剪多档播客，每档风格不同，人员流动大
目标：标准化流程，新人上手快，批量处理

Flow 重点：
· 多播客管理 — 每档节目独立偏好档案
· 模板系统 — "访谈类模板"、"闲聊类模板"、"独白类模板"
· 批量上传 — 一次传多期，排队处理
· 团队协作 — 剪辑师审查 → 主播终审
· 导出格式灵活 — 对接不同平台（小宇宙/苹果/Spotify）
```

## 五、我们先做什么？（MVP 优先级）

```
Phase 1 — 粗剪审查（已有 ✓）
  上传 → 转录 → 粗剪 → 审查页面 → 导出粗剪版
  偏好学习 v1（记录用户修改）

Phase 2 — 高光生成
  从粗剪版自动推荐高光片段
  用户选 3 个 → 自动拼片头预览

Phase 3 — Web App 化
  当前是 Claude Code CLI，需要搬到网页端
  用户 login → 上传 → 在线审查 → 下载成品

Phase 4 — 偏好学习 v2 + hands-free
  跨期学习
  "3 次后 hands-free" 正式上线

Phase 5 — 团队/机构版
  多播客管理
  批量处理
  团队协作审查
```
