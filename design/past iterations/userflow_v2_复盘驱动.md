# User Flow V2 — 复盘驱动的播客剪辑

核心洞察：主播录完都会复盘，这个复盘本身就是最好的剪辑指令。

---

## 完整 Flow

```

  STEP 0 · 节目档案（一次性设置）

  · 风格文档（一段话描述节目调性）
```

## 多主播复盘的冲突处理

```
阿司的复盘："MBTI 那段有点水"
雨林的复盘："MBTI 那段挺好玩的想留"

→ AI 在 Brief 里标记为冲突：
  "阿司觉得可以删，雨林想留。
   你们商量一下？[听这段 ▶]"

→ 谁有最终决定权？
  · 可以设置：主播 A 拥有内容终审权
  · 或者：多数决
  · 或者：剪辑师决定
```

## 节目 Taste Profile 示例

```yaml
节目名: 辟客
风格: |
  三人闲聊 + 嘉宾访谈，氛围轻松但有深度。
  保留自然的笑声、反应和思考停顿。
  不追求极致紧凑，允许适度的展开和跑题。
  但重复叙述和低密度段落应该精简。

主播:
  阿司:
    角色: 主持
    注意: 录音有回声，内容占比不宜过高
  雨林:
    角色: 主持
    注意: 擅长引导话题转换

默认设置:
  目标时长: 50-60 分钟
  删减力度: 适中
  目标平台: 小宇宙
  
学到的偏好（自动更新）:
  - 珊瑚词典类的艺术品描述：保留核心概念，精简逐一举例
  - 教授对话类的场景回忆：保留（用户喜欢这类叙事）
  - 制作讨论：全删
  - MBTI 讨论：用户倾向删除
```

## 与当前 Flow 的关键差异

| | 当前 Flow | 复盘驱动 Flow |
|--|-----------|-------------|
| 第一步 | 上传 → 等结果 → 审查 | 上传 → **复盘** → 等结果 → Brief |
| AI 的输入 | 只有音频 | 音频 + 复盘 + 风格文档 + 历史偏好 |
| 用户决策量 | 800 句逐句判断 | 复盘 5 分钟 + Brief 回答 2-3 个问题 |
| AI 的表现 | "我帮你删了这些" | "你说要留的我留了，你说水的我删了，这 2 个我不确定" |
| 信任感 | 低（AI 自作主张） | 高（AI 在执行你的意图） |
| 多人协作 | 不支持 | 多人复盘 + 冲突标记 |
